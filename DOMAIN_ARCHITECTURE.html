🖥️ Компоненты системы

1. АДМИН-ПАНЕЛЬ (admin.platform.kz или localhost:3000)
  • Next.js 14 App Router
  • Vercel или собственный VPS
  • Функции: Генерация → Мониторинг → QA → Публикация
  • Подключена к единой PostgreSQL БД
2. ФРОНТЕНД ДОМЕНЫ (santehnik.kz, elektrik.kz, и т.д.)
  • Static HTML + CSS (генерируется админ-панелью)
  • Или динамические Next.js страницы
  • Все читают из одной БД
  • SEO-оптимизированные URL: /almaty-remont-trub/, /turgen-remont-trub/
3. ЕДИНАЯ БАЗА ДАННЫХ (PostgreSQL)
  • Хранит весь контент для всех доменов
  • Таблица domains (santehnik.kz, elektrik.kz, ...)
  • Таблица pages (все генерированные страницы)
  • Таблица locations (города)
  • Таблица templates (шаблоны HTML)

🔗 Как подключить домен: 5 шагов

ШАГ 1: Купил домен santehnik.kz 🏪
  • Зарегистрировал у регистратора (Reg.kz, Nameserver и т.д.)
  • Получил доступ к DNS управлению
ШАГ 2: Настроил DNS записи 🔧
Вариант DNS Запись Описание
На Vercel CNAME: platform.vercel.app Если хостишь на Vercel (рекомендуется)
На VPS A: 123.45.67.89 Если у тебя собственный сервер
ШАГ 3: Добавил домен в БД 💾

ШАГ 4: Настроил SSL сертификат 🔐
  • На Vercel: автоматически
  • На VPS: Let's Encrypt (бесплатно)
  • Все домены → https:// обязательно
ШАГ 5: Генерирую контент из админ-панели 🚀
  • Выбираю: Домен = "santehnik.kz", Нишу = "Сантехника"

  • Фронтенд santehnik.kz показывает контент!

🔄 Как работает публикация

1. АДМИН-ПАНЕЛЬ (Генерация) └─→ Выбираешь домен + нишу + города └─→ Создается промпт для Claude API └─→ API генерирует контент └─→ Сохраняется в таблицу pages (domain_id = santehnik.kz) 2. QA ПРОВЕРКА └─→ Берутся страницы из pages для santehnik.kz └─→ Показываются в админ-панели └─→ Ты одобряешь или отклоняешь └─→ Статус меняется на published 3. ПУБЛИКАЦИЯ ВОЛНАМИ └─→ Берутся одобренные страницы └─→ Публикуются волнами по 100 шт каждые 3 дня └─→ Дата публикации сохраняется в БД 4. ФРОНТЕНД ПОКАЗЫВАЕТ └─→ Пользователь идет на santehnik.kz/almaty-remont-trub └─→ Next.js запрашивает БД └─→ Находит страницу где domain='santehnik.kz' и slug='almaty-remont-trub' └─→ Отображает HTML

🚀 Как развернуть: 2 варианта

  • Что включено: Хостинг, SSL, CDN, autoscaling

1. Загрузишь код на GitHub 2. Подключишь Vercel к GitHub репо 3. Vercel автоматически deploy при каждом push 4. В Vercel dashboard добавишь домены 5. Переведешь DNS у регистратора на Vercel DNS: CNAME platform.vercel.app 6. Через 5-10 минут домен работает! 🎉

Запустить с pm2: pm2 start npm --name app -- start 7. Настроить nginx как reverse proxy 8. Получить SSL с certbot: certbot certonly -d santehnik.kz 9. Настроить DNS: A запись на IP сервера 10. Готово!

  • 3. Отобрази content как HTML
5. Развернуть на Vercel
  • Запушить код на GitHub
  • Vercel автоматически deploy
  • Добавить домены в Vercel
  • Обновить DNS у регистратора
  • Готово!

❓ Частые вопросы

Q: Нужен ли отдельный сервер для каждого домена?

A: НЕТ! Один сервер с одной БД может обслуживать 100+ доменов. Это очень масштабируемо и экономно.

Q: Сколько doменов я могу иметь?

A: Столько, сколько захочешь! Система масштабируется на 1000+ доменов без проблем.

Q: Что если один домен упадет в Google?

A: Остальные домены продолжают работать. Это называется "домейн диверсификация" и это отличная защита.

Q: Как быстро будут открываться страницы?

A: Очень быстро! Страницы генерируются один раз и кешируются. Среднее время загрузки: 50-150ms.

Q: Сколько это стоит в месяц?

A: На Vercel: от $20/месяц за админ-панель. Домены: $10-15 каждый (один раз в год). Доступно!
